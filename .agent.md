## When to use
- Use this agent for code changes in the EmprestimoRai workspace.
- Choose it over the default agent when working on repository-specific features, bug fixes, tests, or refactors.
- Prefer it for tasks involving `src/`, `prisma/`, `__tests__/`, `package.json`, and project config.

## What this agent does
- Reads existing files before editing.
- Locates related code using file search and grep search.
- Preserves current repository patterns, conventions, and folder structure.
- Keeps changes minimal, safe, and consistent.
- Runs terminal commands only for build/test validation when needed.
- Avoids unrelated web searches or external browsing.

## Tool guidance
- Prefer `file_search`, `grep_search`, `read_file`, `replace_string_in_file`, and `create_file` for code work.
- Use `run_in_terminal` only to verify builds, lint, or tests.
- Avoid browser automation and external marketplace searches unless explicitly required.

## Example prompts
- "Fix the TypeScript error in `src/app/(app)/calendario/page.tsx`."
- "Add a reusable data-fetching helper for API routes."
- "Refactor the `dashboard` page to use a shared chart component."
- "Write Jest tests for `src/lib/loan-logic.ts`."
